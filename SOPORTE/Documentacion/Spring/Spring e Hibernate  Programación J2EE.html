Spring e Hibernate

By Daniel | 9 agosto 2009
17 comentarios

Hoy os voy a hablar de como integrar correctamente Spring con Hibernate, como todos sabréis Hibernate es un framework de persistencia que nos hace mucho mas fácil el desarrollo de una aplicación con una base de datos.

Para integrar Hibernate con Spring, debemos configurar mediante beans e inyección de dependencias los diferentes objetos requeridos por Hibernate, también debemos configurar las clases java a las que serán mapeadas las tablas de la base de datos. Y por ultimo los diferentes Controllers u otras clases con las que interactuaremos con la base de datos.

Empezaremos, por seguir el orden mas lógico, creando las clases java y sus archivos de mapeo con las tablas de la base de datos.

Imaginaros que tenemos la siguiente estructura de tabla.

CREATE TABLE `usuarios` (
  `usuario` varchar(45) NOT NULL,
  `contrasena` varchar(45) NOT NULL,
  PRIMARY KEY (`usuario`)
)

Su correspondiente clase en java sera la siguiente.

package …;
 
import java.io.Serializable;
 
public class Usuario implements Serializable {
 
    private String usuario;
    private String contrasena;
 
    public String getUsuario() {
        return usuario;
    }
 
    public void setUsuario(String usuario) {
        this.usuario = usuario;
    }
 
    public String getContrasena() {
        return contrasena;
    }
 
    public void setContrasena(String contrasena) {
        this.contrasena = contrasena;
    }
 
}

Y el archivo xml de Hibernate que relacionara la tabla con nuestra clase java tendrá por nombre “Usuario.hbm.xml” y sera así.

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE hibernate-mapping PUBLIC "-//Hibernate/Hibernate Mapping DTD 3.0//EN" "http://hibernate.sourceforge.net/hibernate-mapping-3.0.dtd">
<hibernate-mapping>
  <class name="...Usuario" table="usuarios">
    <id column="usuario" name="usuario" type="string"/>
    <property column="contrasena" name="contrasena" not-null="true" type="string"/>
  </class>
</hibernate-mapping>

Como veis hay que especificar la version xml y su codificación, y aparte el doctype. Aparte de esto, el atributo name dentro de class especifica la clase Java a la que hace referencia, y el atributo table el nombre de la tabla en la base de datos. Después, están mapeadas cada una de las columnas, id para el identificador de la tabla, y property para el resto de columnas, cada columna tiene establecidas sus atributos, los obligatorios son column, el nombre de la columna en la tabla de la base de datos, el name, el nombre del atributo en nuestra clase Java (no tienen porque coincidir column y name), y el type, el tipo de dato Java. El resto de propiedades dependerán de las propiedades que tenga la columna en la tabla de la base de datos.

Ya tenemos mapeada nuestra clase Java con su xml en la base de datos. El siguiente paso sera crear los beans para configurar la base de datos, nos harán falta.

  • Un DataSource: Un DataSource es un objeto JNDI (Java Naming and Directory Interface) con el cuál obtendremos una conexión desde un Connection Pool hacia una base de datos.
  • Un SessionFactory: Es el responsable de crear la sesión a través de la cual Hibernate realizara las transacciones y accesos de datos pertinentes.

Para configurar el DataSource lo unico que tenemos que hacer es crear el bean, especificandole el id que le queramos dar y el tipo de class con el que queremos crear el DataSource. Despues, especificamos las cuatro propiedades necesarias para el acceso a la base de datos y ya esta, asi de simple. Como en mi caso estoy utilizando una base de datos MySQL el driver que utilizare puede diferir del que debéis utilizar vosotros según la base de datos que estáis usando.

<bean id="miDataSource" class="org.apache.commons.dbcp.BasicDataSource" destroy-method="close">
    <property name="driverClassName" value="com.mysql.jdbc.Driver"/>
    <property name="url" value="jdbc:mysql://host:port/nombredelabasededatos"/>
    <property name="username" value="vuestrousuario"/>
    <property name="password" value="vuestracontraseña"/>
</bean>

NOTA: La clase “org.apache.commons.dbcp.BasicDataSource” necesita las librerías “commons-dbcp-version.jar” y “commons-pool-version,jar” del proyecto Apache para funcionar.

Para configurar el SessionFactory debemos establecer su id, su class, como en cualquier bean, y ademas tres propiedades. La primera es el DataSource, que ya lo hemos configurado previamente, simplemente lo referenciamos y listo, lo siguiente son los distintos archivos de mapeo que vayamos a utilizar, en nuestro caso también lo hemos creado ya, y por ultimo la propiedad hibernateProperties, en la que se configurara nuestra SessionFactory. Yo utilizo la librería C3p0 para el pool de conexiones.

<bean id="miSessionFactory" class="org.springframework.orm.hibernate3.LocalSessionFactoryBean">
    <property name="dataSource" ref="miDataSource"/>
    <property name="mappingResources">
        <list>
            <value>.../Usuario.hbm.xml</value>
        </list>
    </property>
    <property name="hibernateProperties">
            <props>
                <!-- Esta propiedad establece que solo se actualiza una tabla cuando haya cambios -->
                <prop key="hibernate.hbm2ddl.auto">update</prop>
                <!-- Esta propiedad establece el lenguaje de la base de datos, en este caso MySQL -->
                <prop key="hibernate.dialect">org.hibernate.dialect.MySQLDialect</prop>
                <!-- Esta propiedad sirve para aplicar un alias en nuestras consultas.
                Si creamos una consulta con un true, nosotros al aplicarle un alias 'T',
                con poner T automaticamente lo recombertira Hibernate en la consulta final
                por nosotros. -->
                <prop key="hibernate.query.substitutions">true 'T', false 'F'</prop>
                <!-- Sirve para desactivar el debug de las consultas en la consola. -->
                <prop key="hibernate.show_sql">false</prop>
                <!-- Especificamos el valor minimo del pool de conexiones -->
                <prop key="hibernate.c3p0.minPoolSize">5</prop>
                <!-- Especificamos el valor maximo del pool de conexiones -->
                <prop key="hibernate.c3p0.maxPoolSize">20</prop>
                <!-- El tiempo de vida de cada conexion del pool. -->
                <prop key="hibernate.c3p0.timeout">600</prop>
                <!-- Especificamos el valor maximo de transferencias -->
                <prop key="hibernate.c3p0.max_statement">50</prop>
                <!-- Espeficica que no chequee el estado de la conexion antes de cada consulta -->
                <prop key="hibernate.c3p0.testConnectionOnCheckout">false</prop>
            </props>
    </property>
</bean>

NOTA: Al configurar las propiedades que tendra SessionFactory con la librería c3p0, implica que lalibrería “c3p0-version.jar”, deba de estar importada en el proyecto.

Con estos simples dos pasos ya tenemos configurado nuestro Hibernate para funcionar en Spring. Ahora pasemos a realizar una pequeña consulta y una pequeña inserción de datos.

Como la consulta va a ser muy simple, creare un objeto hibernateTemplate, que nos suministra Spring, y que únicamente necesita nuestro SessionFactory para funcionar y realizar una consulta simple, aunque en una de las próximas entradas os enseñare como realizar la consulta a través de objetos Criteria, los cuales nos permiten realizar consultas mucho mas avanzadas.

Nuestras clases para consultar e insertar datos.

public class ConsultaController implements Controller {
 
    private HibernateTemplate hibernateTemplate;
 
    public ModelAndView handleRequest(HttpServletRequest request, HttpServletResponse response)
            throws ServletException, IOException {
 
        Usuario usuario = new Usuario();
 
        String query = "from Usuario where usuario = ?";
        /* Este es un ejemplo simple y no depurado, por eso necesitamos que exista un usuario llamado “pedro” en nuestra base de datos para que no de error, el .get(0) es debido a que hibernateTemplate nos devuelve una list, en este caso yo se que unicamente existe un usuario pedro, asi que por simplificar el ejemplo lo he recogido de la list directamente.*/
        usuario = (Usuario) hibernateTemplate.find(query, "pedro").get(0);
 
        return new ModelAndView("mivista", "usuario", usuario);
    }
 
    public void setSessionFactory(SessionFactory sessionFactory) {
        this.hibernateTemplate = new HibernateTemplate(sessionFactory);
    }
}
public class GuardaController implements Controller {
 
    private HibernateTemplate hibernateTemplate;
 
    public ModelAndView handleRequest(HttpServletRequest request, HttpServletResponse response)
            throws ServletException, IOException {
 
        Usuario usuario = new Usuario();
        /* request.getParameter("") recoge los parámetros recibidos por el cliente tanto por GET como por POST*/
        usuario.setUsuario(request.getParameter("usuario"));
        usuario.setContrasena(request.getParameter("contrasena"));
 
        hibernateTemplate.save(usuario);
 
        return new ModelAndView("mivista");
    }
 
    public void setSessionFactory(SessionFactory sessionFactory) {
        this.hibernateTemplate = new HibernateTemplate(sessionFactory);
    }
}

Ahora lo único que nos falta es crear los beans de nuestras clases DAO inyectandoles nuestro SessionFactory.

<bean id="consultaController" class="...ConsultaController">
    <property name="sessionFactory" ref="miSessionFactory" />
</bean>
 
<bean id="guardaController" class="...GuardaController">
    <property name="sessionFactory" ref="miSessionFactory" />
</bean>

Y mapearlos en nuestro urlMapping.

<bean id="urlMapping" class="org.springframework.web.servlet.handler.SimpleUrlHandlerMapping">
    <property name="mappings">
        <props>
            <prop key="consulta.htm">consultaController</prop>
            <prop key="guarda.htm">guardaController</prop>
        </props>
    </property>
</bean>

    Hola Julito, bienvenido.

    El objeto “usuario” creado en las clases “ConsultaController” y “GuardaControler” es una lista (java.util.List), no una clase, ya que es el tipo de objeto que devuelve el método “.findByCriteria” del HibernateTemplate, esta List después, se la envío al JSP donde la manejo con JSTL Core. En esta List es donde se almacenan los diferentes resultados de la búsqueda en cuestión sobre la base de datos.

    Los objetos servidos a un “Controller” mediante “inyección de dependencias” están declarados como un o un en el bean de dicho “Controller”, es ahí, y solo ahí, donde se le inyectan los distintos objetos a una clase.

    La librería utilizada por “miSessionFactory” es la propia de Hibernate 3, pero para crear este objeto que permite una mejor integración con Spring, Spring nos ofrece diferentes clases para hacerlo, “org.springframework.orm.hibernate3.LocalSessionFactoryBean” es una de ellas, y forma parte de las librerías propias de Spring.

    He actualizado las propiedades del “hibernateProperties” con una explicación de cada valor.

    Un saludo!.
